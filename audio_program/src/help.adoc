=== Language and stack model

Sound Garden programs are whitespace-separated concatenative words. Each word consumes zero or more frames from the stack and pushes zero or more frames back. A frame contains one value per audio channel; scalar constants are broadcast to all channels. Stack effect notation below uses the logical order `(older, ..., top) -> result`.

Unknown words, stack underflow, and stack overflow are tolerated by the VM and effectively produce default/ignored values rather than aborting audio.

[horizontal]
return, ret, !:: stop compiling/evaluating the program here; the current stack/audio output is used and all following words are ignored

=== Comments

Any word starting with `;` is ignored by the compiler, e.g. `;bassline` or `;slow-attack`. Comments are single words (use dashes inside one comment word, or several `;` words); this matches the editor where every node is one word.

=== Reproducibility

[horizontal]
seed:<N>:: compile-time directive (consumes nothing, produces nothing): seeds all random generators — `noise`, spectral transforms, and pattern random choice `|` — so renders are reproducible. Without it every run is unique.

=== Constants and literals

[horizontal]
<number>:: push a numeric constant, e.g. `0.5`, `-12`, `1e-3`
<numerator>/<denominator>:: push a ratio literal with numeric numerator and denominator and no spaces, e.g. `5/4`, `3/2`, `1.5/4`
silence:: () -> push constant 0.0
sr:: () -> push current sample rate
pi:: () -> push π
tau:: () -> push 2π
whiteNoise, noise, n:: () -> pseudo-random white noise; each sample/channel receives the next generator value

Scientific pitch notation is also available using MIDI note 60 = C4. Lowercase notes push frequencies (`c4` = 261.625565, `a4` = 440); uppercase notes push MIDI note numbers (`C4` = 60, `A4` = 69). Sharps and flats are supported (`c#4`, `Db4`).

=== Stack manipulations

[horizontal]
pop:: (a) -> remove top element
dup:: (a) -> a a
swap:: (a, b) -> b a
rot:: (a, b, c) -> b c a
dig:<N>, :<N>:: take Nth element from the top and put it on the top

=== Oscillators and phases

Oscillator phases are wrapped in the `-1..1` range unless noted otherwise. Oscillator outputs are in the `-1..1` range.

[horizontal]
saw:: (freq, phase0) -> saw/phasor oscillator with explicit phase offset
w:: (freq) -> saw/phasor oscillator with phase0 = 0
tri:: (freq, phase0) -> symmetric triangle oscillator with explicit phase offset
t:: (freq) -> triangle oscillator with phase0 = 0
pulse:: (freq, width, phase0) -> rectangular oscillator; `width` is the positive segment as a ratio of period
p:: (freq, width) -> pulse oscillator with phase0 = 0
sine:: (freq, phase0) -> sine oscillator with explicit phase offset
s:: (freq) -> sine oscillator with phase0 = 0
cosine:: (freq, phase0) -> cosine oscillator with explicit phase offset
c:: (freq) -> cosine oscillator with phase0 = 0
cycle, cy:: (cps) -> wrapped `0..1` phase, advanced by cycles-per-second input; negative CPS runs backwards

Fast approximate variants are available for sine/cosine oscillators and trig functions: `s'`, `sine'`, `c'`, `cosine'`, `sin'`, `cos'`, `tan'`, `sinc'`.

=== Math

Binary arithmetic operations are available: `+`, `-`, `*`, `/`, `%`, `^`. Aliases are `add`, `sub`, `mul`, `div`, `mod`, `pow`. Division and reciprocal are safe: division by zero produces 0.0.

The backslash op computes reciprocal; for example `16 \` produces `1/16`.

[horizontal]
min:: (a, b) -> pairwise minimum
max:: (a, b) -> pairwise maximum
clamp:: (x, min, max) -> clamp x to the provided range
clip:: (x) -> clamp x to -1..1
wrap:: (x) -> wrap x around the -1..1 range
exp:: (x) -> e^x
sin:: (x) -> sine of x radians
cos:: (x) -> cosine of x radians
tan:: (x) -> tangent of x radians
sinc:: (x) -> sin(x) / x, with sinc(0) = 1
sinh:: (x) -> hyperbolic sine
cosh:: (x) -> hyperbolic cosine
tanh:: (x) -> hyperbolic tangent
round:: (x) -> round to nearest integer
quantize, q:: (x, step) -> round x to the nearest multiple of step
linlin, project:: (x, a, b, c, d) -> linearly project x from range a..b to range c..d
linexp:: (x, a, b, c, d) -> linearly project x from range a..b to exponential range c..d
explin:: (x, a, b, c, d) -> exponentially project x from range a..b to linear range c..d
expexp:: (x, a, b, c, d) -> exponentially project x from range a..b to exponential range c..d

=== Patterns

Patterns are signal-native cycle readers for direct musical development over time. Numeric patterns are comma-separated and accept the same scientific pitch constants as programs (`C4` = 60, `c4` = 261.625565). Gate and trigger patterns use dense visual notation where `x`/`X` is active and `.` is inactive; ASCII whitespace and commas are ignored.

Bracketed groups subdivide one pattern cell without needing whitespace, e.g. `gate:x[x.]..` gives the second top-level cell two half-speed subcells, and `pat:60,[64,67],72,67` plays 64 then 67 inside the second cell. Groups can be nested. `*N` repeats the preceding atom or group N times, e.g. `gate:[x.]*4` or `pat:60*2,[64,67]*2`. Event suffixes support Euclidean rhythms with `(PULSES,STEPS)`, e.g. `gate:x(3,8)` or `pat:60(3,8)`; numeric Euclidean rests default to `0` and can be set with `(PULSES,STEPS,OFF)`, e.g. `pat:60(3,8,-12)`. `gate:e(3,8)` is also accepted. Alternation uses `<A;B;...>` anywhere in a pattern and advances on each forward cycle wrap, e.g. `gate:<x.;.x>`, `gate:x[<x.;.x>]x`, or `pat:60,<64;67>,72`. Random choice uses `|` between alternatives, e.g. `gate:x|.` or `pat:60|64`. In numeric patterns, `_` holds the previous value; leading holds wrap to the last value in the cycle.

[horizontal]
pat:<PATTERN>:: (phase) -> read a numeric pattern using wrapped `0..1` phase, e.g. `pat:60,64,67,72`, `pat:C4,E4,G4,c5`, `pat:60,[64,67],72`, `pat:60*2,64`, `pat:60,_,64,_`, `pat:60(3,8)`, `pat:60(3,8,-12)`, `pat:<60,64;67,72>`, or `pat:60|64`
gate:<PATTERN>:: (phase) -> held gate from a dense pattern, e.g. `gate:x..x`, `gate:x[x.]..`, `gate:[x.]*4`, `gate:x(3,8)`, `gate:<x.;.x>`, or `gate:x|.`
trig:<PATTERN>:: (phase) -> one-sample trigger on entering an active cell, e.g. `trig:x..x`, `trig:x[xx]..`, `trig:[x.]*4`, `trig:x(3,8)`, `trig:<x.;.x>`, or `trig:x|.`
cpat:<PATTERN>:: (cps) -> convenience form of `cycle` followed by `pat:<PATTERN>`
cgate:<PATTERN>:: (cps) -> convenience form of `cycle` followed by `gate:<PATTERN>`
ctrig:<PATTERN>:: (cps) -> convenience form of `cycle` followed by `trig:<PATTERN>`

=== Polyphony

A bracket group immediately before `poly` is a quotation: it is compiled into a voice body instead of being spliced as a template. `poly:N` runs N copies of the body and sums their outputs. On a rising edge of the control signal it allocates the next voice (round-robin) and latches the current value into it; the most recently allocated voice receives the live control signal as-is (a trig body sees a one-sample impulse, a gate body sees the full gate including its fall, so `adsr` releases work), other voices receive 0. Control amplitude passes through, so gates can carry velocity. Each voice body starts from a stack of `(value, ctl)`. Named templates work as bodies too: `lead poly:4`.

[horizontal]
poly:<N>:: (value, ctl) -> sum of N voices of the preceding quotation, e.g. `1 cycle pat:60,64,67,72 m2f 1 cycle trig:x.xx [ swap s swap 0.01 impulse * ] poly:4`

[horizontal]

oneshot, shot:: (period) -> emit 1.0 once after the given period in seconds, 0.0 otherwise

[horizontal]

=== Spectral transforms

All spectral transforms use a 2048-sample Hann window with a 64-sample hop and proper overlap-add synthesis; they add about one window (~43 ms at 48 kHz) of latency. Random spectral ops respect `seed:<N>`.

[horizontal]
spectral_shuffle:<N>:: (x, amount, trig) -> shuffle frequency bins: a new permutation is drawn on each trigger (and once at start) over the `amount` (0..1) fraction of bins; with <N> bins may only move within blocks of N neighbours (default global). `noise 1 1 spectral_shuffle` re-scrambles every hop; `... 0.7 1 dmetro spectral_shuffle:16` re-draws a local scramble each second
spectral_freeze:: (x, gate) -> while the gate is high, sustain the magnitude spectrum captured at the gate's rise with randomized phases; while low, pass through
spectral_reverse:: (x) -> reverse frequency bins
st1:: (x) -> keep only the strongest bin and zero the rest

=== Tables

[horizontal]
writetable:<NAME>:<N>, wtab:<NAME>:<N>, wt:<NAME>:<N>:: (x, trigger) -> on trigger write N seconds, per channel, of signal x to table NAME; x is passed through
readtable:<NAME>, rtab:<NAME>, rt:<NAME>:: (indexer) -> read from table NAME using indexer signal as a position in seconds, with linear interpolation
filetable:<FILE>, ftab:<FILE>, ft:<FILE>:: (indexer) -> read from the table loaded from FILE using indexer signal as a position in seconds, with linear interpolation. Supported formats: WAV, FLAC, OGG.
